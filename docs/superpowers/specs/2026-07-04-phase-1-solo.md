# Phase 1 — Solo Teacher Onboarding Wizard

**Date:** 2026-07-04
**Status:** Spec ready, ready for swarm dispatch
**Owner:** Coder agent (assigned per swarm dispatch)
**Depends on:** Phase 0 (shipped — `educational_assistant` role in DB)

## Goal

A solo teacher can sign up, complete a 5-step onboarding wizard that ends in their first scheduled duty + reminder, in under 2 minutes — without admin involvement, school setup overhead, or seeing features they don't have access to yet.

- `routes/signup.tsx` — `?mode=solo` opens the solo card by default
- `routes/onboarding.admin._index.tsx` — solo-path blurb on admin completion screen
- `packages/db/src/schema.ts` — `userRoleEnum` includes `'educational_assistant'`
- Migration `0007_user_role_educational_assistant.sql` — applied to live DB
- The `/api/signup/solo` flow exists and works (creates school + user with role `school_admin`)

## What Phase 1 needs to add

### 1.1 Solo role selection during signup

**Why:** The current `signupSolo` flow always creates the user as `school_admin`. That's correct for "create your own school" but a teacher-only path should let the user identify as a teacher (or EA) and start with the right role + the right onboarding.

**File changes:**
- `apps/web/server/signup.server.ts` — extend `signupSolo` to accept optional `role: 'teacher' | 'educational_assistant' | 'school_admin'`. Default `school_admin` for backward compatibility. Validate against the new enum.
- `apps/web/app/routes/api.signup.solo.ts` — pass the `role` form field through.
- `apps/web/app/components/SignupCard.tsx` — solo card gets a role picker (radio: Teacher / Educational Assistant / School admin). Default to Teacher for solo flows.
- Migration 0008: if role is provided, write to `users.role` (already supported by the schema). No new SQL needed — just confirm the constraint allows the new role values (it does).

**Acceptance:**
- A teacher signs up via solo flow → their `users.role` is `teacher`, they get redirected to `/onboarding/teacher`.
- An EA signs up via solo flow → `users.role` is `educational_assistant`, redirected to a new `/onboarding/ea` flow.
- A solo admin (current behavior) → `users.role` is `school_admin`, redirected to `/onboarding/admin`.

### 1.2 Solo onboarding wizard (new route)

**Route:** `apps/web/app/routes/onboarding.solo._index.tsx` — replaces direct redirect to `/app/today` for solo users.

**5 steps, one card per step (mirrors admin wizard style):**
1. **Welcome** — "Hi {name}. Let's get your supervision schedule set up in 4 short steps."
2. **School district** — single select: Ontario (TDSB, YRDSB, PDSB, OCDSB, other) / BC / Alberta / Other / "I'm outside North America". Solo users skip the school-code step from admin onboarding.
3. **Cycle length** — radio: 5 days (most common) / 6 days / Custom. Default 5.
4. **First duty** — minimal form: name (placeholder: "Morning recess"), location (placeholder: "Front doors"), start time, end time. Save creates one `duty` row + one `dutyAssignment` row for the user.
5. **Reminder style** — radio: "15 min before, email only" (default) / "30 min before, email + SMS if I give my phone" / "Custom (later in Settings)".

**File changes:**
- `apps/web/app/routes/onboarding.solo._index.tsx` — new route, copy the admin wizard's progress-dot style.
- `apps/web/app/routes/api.onboarding.solo.ts` — new action route that takes the wizard's submitted data and writes `duty` + `dutyAssignment` rows. CSRF-protected.
- `apps/web/app/components/onboarding/SoloStep*.tsx` — extract each step into its own component if the inline JSX exceeds 50 lines.

**Acceptance:**
- Solo user goes through all 5 steps, lands on `/app/today` with one scheduled duty visible.
- Each step has a "Back" button (except step 1) and a "Next" / "Finish" button (except step 5).
- Skipping steps (browser refresh) doesn't lose state — each step writes to the DB before showing the next, OR state is preserved in the URL.

### 1.3 Onboarding gates in `/app/today`

**Why:** Without a wizard, solo users land on Today with zero duties and see a confusing "no cycle info" placeholder. With the wizard, they should never see Today empty — but as a fallback for users who skip the wizard, add a "Set up your first duty" banner.

**File changes:**
- `apps/web/app/routes/_app.today._index.tsx` — if the user's `dutyAssignments` count is 0 AND they have no `schools.demoExpiresAt`, render a yellow banner at the top: "Welcome — let's add your first duty" → links to `/onboarding/solo` (with a "skip for now" button to dismiss).
- The banner persists until the user has at least one duty.

**Acceptance:**
- New solo user lands on Today after the wizard — banner is hidden because they have a duty.
- New solo user who skipped the wizard — banner shows with a CTA to `/onboarding/solo`.

### 1.4 EA-specific reminder defaults

**Why:** EAs in Jason's table cover specific slots in the cycle. They don't "complete" duties the same way teachers do — they show up, cover, leave. The Mark Complete flow should be hidden for EAs.

**File changes:**
- `apps/web/app/routes/_app.today._index.tsx` — if `role === 'educational_assistant'`, hide the "Mark complete" button on duty cards.
- `apps/web/app/components/.../DutyCard.tsx` — same condition; swap to a passive "Covering" badge instead of the active button.
- `apps/web/app/components/.../EquipmentChips.tsx` (or wherever Mark Complete lives) — same.
- `apps/web/app/routes/app.api.duty.complete.ts` — server-side check: refuse the request if `role === 'educational_assistant'` (return 403).

**Acceptance:**
- EA logs in, sees their duty card with no Mark Complete button.
- EA's POST to `/app/api/duty/complete` returns 403 with message "EAs use the coverage flow instead."

### 1.5 Demo seed updates

**Why:** The demo school must include at least one EA so new visitors can see the EA UX.

**File changes:**
- `apps/web/server/demo-seed.server.ts` — add 1 EA user to the Sunrise Elementary seed. Pick a slot from Jason's actual table — "Cyriac, Loganathan, Sheikh" → make one of them an EA, the others teachers.

**Acceptance:**

- `/app/today` for the EA account shows duty cards with the EA-specific UX.
